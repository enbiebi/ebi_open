    <head>

            body{

    </head>

<p>JavaScript can change the content of an HTML element:</p>

<button type="button" onclick="myFunction()">Click Me!</button>

meta要素について

文字コードの指定のほかにも、いろいろなことができる

検索エンジンのキーワード

<meta name="keywords" content="HTML, CSS, JavaScript">

ページの作成者を定義

<meta name="author" content="John Doe">

30秒ごとに更新

<meta http-equiv="refresh" content="30">

ビューポートを設定して、すべてのデバイスでWebサイトが適切に表示されるようにする

<meta name="viewport" content="width=device-width, initial-scale=1.0">
initial-scale=1.0:ブラウザーによってページが最初に読み込まれるときの初期ズームレベルを設定

ビューポートの設定は必須

max-widthとvm

mac-width

  • width: 100%; - 画面幅いっぱいに表示。幅に合わせて画像が拡大縮小される。
  • max-width: 100%; - (画面の大きさ)＜(画像の大きさ)なら縮小。でなければ、元の画像サイズで表示させる。

heightはautoにしておこう

vm

ビューポートの幅を意味する。

Hello World

h4 style="font-size:10vw"

1vw = ビューポートの幅の1%
ビューポートの幅が50cmの場合、1vwは0.5cm

URL

URL=Uniform Resource Locator

scheme://prefix.domain:port/path/filename

  • scheme（スキーム）：インターネットサービスの種類を定義します（一般的なのは http や https）。
  • prefix（プレフィックス）：ドメインの接頭辞を定義します（http の場合、通常は www）。
  • domain（ドメイン）：インターネット上のドメイン名を定義します（例：w3schools.com）。
  • port（ポート）：ホスト上のポート番号を定義します（http のデフォルトはポート 80）。
  • path（パス）：サーバー上のパスを定義します（省略された場合はサイトのルートディレクトリを指します）。
  • filename（ファイル名）：ドキュメントやリソースの名前を定義します。
スキームの種類と用途
スキーム（Scheme） 略語 用途
http HyperText Transfer Protocol 一般的なウェブページに使用。暗号化されていない
https Secure HyperText Transfer Protocol 安全なウェブページに使用。暗号化されている
ftp File Transfer Protocol ファイルのダウンロードやアップロードに使用
file ― 自分のコンピューター上のファイルにアクセスするために使用
